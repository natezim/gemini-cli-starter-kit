# Project Context

This file is written by /start or /setup based on your project.
Run /start to generate it automatically, or write it yourself.
Update anytime with /context:update.

Tip: Drop reference files (specs, docs, schemas) into ./context/ — they're
auto-loaded every session and used to generate smarter setup questions.

WARNING: Files in context/ are loaded into every Gemini session automatically.
NEVER put credentials, .env files, service account keys, or PII in this folder.
